Our Solution

After lots of trial and error, we have found that Esri's ArcGIS Experience Builder is the best technology for a user-friendly public web application that will help people living and visiting Lindsay, Ontario, navigate around the city via public transit. Here's our methodology to create our final solution:

Step 1: Create an experience

Create an Experience using the ArcGIS Online component from the menu button. Click the Experience Builder app.

Create a new Experience using the "Create New" button on the top righthand side.

There will be many templates presented to you. Choose wisely for the application of your interest. In our case, we are creating a Transit application for Lindsay, so this application should be Map centric.

Step 2: Add data

Click on the map in the center of the screen, then add data from the "Select map" tab on the righthand side. This enables users to add data from either ArcGIS Online or a local database.

After adding the data, you will see the stops and routes of the Lindsay transit in the application. This routes features are what we created from the GTFS and Ontario GeoHub transit data.

NOTE: Don't use the routing widget from the Experience builder as this requires credits and request per head. This is a useless function on this point.

Step 3: Embed ArcGIS Dashboard

Using an "Embed" widget, Sharmila's ArcGIS Dashboard was embedded into the experience via a URL. You may be prompted to sign in to ArcGIS Online to view the Dashboard.

Step 4: Add splash page

In the editing interface of the Experience Builder, click the page icon on the upper lefthand side of the sidebar. Click Window to switch to viewing all the windows within the experience. Click the + icon on the window to add a window.

Scroll down and choose your layout of choice of this window. I chose the Simple version. Give it a name and then click "Set as splash" icon next to the name.

Add an image and text. On our splash page, we wanted to include an image of the routes and also text indicating what the purpose of the experience is. Since this experience was for exploration purposes and the data needs some fine tuning, we added a disclaimer at the bottom. User must accept the terms and conditions in order to view the experience. Customize the user experience with the splash page via the window on the righthand side.

Now, when the experience is visited, this splash page will open up. If the user does not accept terms and conditions, the checkbox will flash red, and they will not be able to see the experience until they click it. If the user does accept the terms and conditions, they will see the experience.

Step 5: Add the Near Me widget

Click and drag the map centric "Map" widget to the experience interface and connect it to the Lindsay map added earlier. Then click and drag the map centric "Near Me" widget to the experience interface and connect it to the map that was just added.

Customize the Near Me widget using the settings below.

Arrange both widgets where you see fit.

Step 6: Review the Solution

After reviewing the solution, it was decided that the Dashboard was redundant. It was visually appealing, but limited the interaction the user can have with the experience. It also made the page look crowded. Therefore, it was removed. The tables from the Dashboard were added to the remaining space. This includes the table with Lindsay fare information and bus route schedules.

TAKE AWAYS

Advantages Disadvantages
Many templates to choose from Can be time consuming to create
Tied to AGOL organization account Moving items around the page can be finicky
Can customize the theme/look Cannot easily move the published experiences between accounts
No coding/programming involved Developed and maintained by Esri, which means the creator has to rely on Esri for updates, support, and compatibility with new technologies
It will automatically update for different screen sizes The data needs to be updated - the bus stops do not match the routes and the scheduled times are not accurate
Can integrate with other apps, like Dashboards and Surveys
Can add triggers that activate depending on how the user interacts with the experience
User can interact with the page using the zoom features on the map, search features on the tables, and location feature on the near me widget

Although there are some tweaks that can make the experience better, it was decided that this was a suitable web solution for the issue we are trying to solve.
